• HTML/JSP: Grade 4. All content is in divs and spaced using bootsrap and HTML5 elements, including datepicker and email are used.
  • CSS: Grade 4. A consistent Bootstrap and custom CSS where used on all display pages.
  • Spring: Grade 4. EmployeeDBDao is autowired for creation at startup and Web.xml and EmployeeController -servlet are configured properly
  • MVC: Grade 4. Employee objects are created to interchange between components.
  • JSTL: Grade 3. For loops are used as, for example, in displaying lists of employees and employee IDs
  • SQL: Grade 3. Full CRUD. User can add employee(C), search for employee(R), update(U), and delete(D).
  • JDBC: Grade 3. SQL Exceptions are handled via try/catch. Resources are closed inside each method calling them
  • Aesthetic: Grade 4. Attempt was made to keep the look modern while keeping in mind the use may be an internal human resources department. Icons and words were used to keep interface intuititive. Placeholder fields were used in addition to labels to guide the user.
  • OOP: Grade 4. Polymorphism is used to call EmployeeDao (the interface) and have EmployeeDBDao used at runtime. No arg constructors are used for the employee and sets are used for the control object in the controller.
